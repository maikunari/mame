opus found an issue with the models calling claude code : Two issues in the systemd service file (/etc/systemd/system/mame.service):

Issue 1: PATH
Systemd sanitizes the PATH, so claude at /usr/bin/claude might not be found. The code already supports an override via MAME_CLAUDE_BIN.

Issue 2: ReadWritePaths
The service only has write access to ~/.mame and ~/Projects/mame. Claude Code needs to write to ~/Projects/sonicpixel-io too. Right now it would crash on permission denied even if the binary was found.

You need to edit the service file. SSH into TH50 and run:

sudo nano /etc/systemd/system/mame.service


Add this line under the existing Environment=MAME_PERSONA=mike:

Environment=MAME_CLAUDE_BIN=/usr/bin/claude


And change this line:
ReadWritePaths=/home/jerry/.mame /home/jerry/Projects/mame

to:
ReadWritePaths=/home/jerry/.mame /home/jerry/Projects


Then reload and restart:
sudo systemctl daemon-reload
sudo systemctl restart mame

Issue 1: ReadWritePaths IS active.

sudo nano /etc/systemd/system/mame.service

ReadWritePaths=/home/jerry/.mame /home/jerry/Projects

Environment=MAME_PERSONA=mike MAME_CLAUDE_BIN=/usr/bin/claude PATH=/usr/local/sbin:/usr/local/bin:/usr/bin:/usr/sbin:/sbin:/bin
